## 工作区概念

Claude Code Pro 使用**工作区**来管理你的代码项目。

### 三种工作区概念

| 概念 | 说明 |
|------|------|
| **当前工作区** | AI 的主要工作目录，所有写入操作都在这里进行 |
| **关联工作区** | AI 可以同时访问的其他工作区，支持跨项目读写操作 |
| **查看工作区** | 文件浏览器当前显示的工作区，可以是任意工作区 |

### 工作区操作

- 在设置中创建新工作区，选择你的项目目录
- 点击工作区名称可以切换当前工作区
- 使用关联工作区功能，让 AI 同时操作多个项目

## 文件引用语法

使用 `@` 符号引用工作区中的文件，让 AI 读取文件内容。

### 当前工作区文件引用

引用当前工作区的文件，使用 `@/` 前缀：

```
@/src/App.tsx
@/components/Button.tsx
@/package.json
```

### 跨工作区文件引用

引用其他工作区的文件，使用 `@工作区名:` 前缀：

```
@utils:src/helpers/format.ts
@frontend:components/Header.tsx
@backend:api/routes.ts
```

### 实际使用示例

```
// 引用并解释文件
@/src/App.tsx 请解释这个文件的入口逻辑

// 跨工作区引用
@utils:src/helpers/format.ts 和 @frontend:src/utils/format.ts 有什么区别？

// 多文件引用
查看 @/src/App.tsx 和 @/src/main.tsx 的关联关系
```

## 斜杠命令

使用 `/` 前缀快速执行常用操作。

### 查看帮助

| 命令 | 说明 |
|------|------|
| `/help` | 显示帮助信息 |
| `/commands` | 列出所有可用命令 |

### Git 操作

| 命令 | 说明 | 示例 |
|------|------|------|
| `/commit` | 创建提交 | `/commit 修复登录bug` |
| `/diff` | 显示修改差异 | `/diff` |
| `/push` | 推送到远程 | `/push main` |
| `/pull` | 拉取远程更新 | `/pull main` |
| `/branch` | 切换/创建分支 | `/branch feature/login` |
| `/status` | 显示 Git 状态 | `/status` |
| `/log` | 显示提交日志 | `/log` |

### 代码质量

| 命令 | 说明 | 示例 |
|------|------|------|
| `/format` | 格式化代码 | `/format` |
| `/lint` | 代码检查 | `/lint` |
| `/test` | 运行测试 | `/test` |
| `/build` | 构建项目 | `/build` |
| `/review` | 代码审查 | `/review` |

### 文件操作

| 命令 | 说明 | 示例 |
|------|------|------|
| `/edit` | 编辑文件 | `/edit @/src/App.tsx 添加错误处理` |
| `/search` | 搜索内容 | `/search useEffect` |
| `/find` | 查找代码 | `/find Button` |

### 上下文管理

| 命令 | 说明 |
|------|------|
| `/context` | 显示当前上下文 |
| `/forget` | 清除指定上下文 |
| `/reset` | 重置会话 |

## 使用示例

### 场景 1：跨项目代码审查

```
请审查 @backend:api/auth.ts 和 @frontend:src/pages/Login.tsx 的登录逻辑，指出潜在的安全问题。
```

### 场景 2：代码重构

```
@/src/components/UserList.tsx 需要重构，请参考 @utils:src/helpers/format.ts 的格式化函数，优化日期显示逻辑。
```

### 场景 3：Git 提交

```
/commit 修复用户登录超时问题
```

### 场景 4：运行测试

```
/test @/src/tests/auth.test.ts
```

### 场景 5：代码解释

```
@/src/App.tsx 这个项目的入口文件结构是什么？请详细解释。
```

## 快捷键

| 快捷键 | 功能 |
|--------|------|
| `Enter` | 发送消息 |
| `Shift + Enter` | 换行 |
| `/` | 触发命令建议 |
| `@` | 触发文件/工作区引用 |
| `Esc` | 关闭建议列表 |
| `↑ / ↓` | 在建议列表中导航 |

---

## 小贴士

1. **自动补全**：输入 `@` 或 `/`后会自动显示建议列表
2. **多文件引用**：一条消息中可以引用多个文件
3. **跨项目操作**：使用关联工作区功能，让 AI 同时操作多个项目
4. **写入位置**：所有写入操作都会在当前工作区执行
